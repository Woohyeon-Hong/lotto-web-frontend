# UI/UX Overview

이번 미션은 기존 CLI 기반 로또 프로그램을 웹 애플리케이션으로 재구현하며 **UI/UX 설계를 직접 수행하는 프로젝트**이다.  
그동안 기능 구현에만 집중하던 개발 방식에서 벗어나, **사용자의 경험을 고려한 인터페이스를 설계**함으로써 사고의 폭을 확장하고자 한다.  
특히 디자인이라는 개인적 약점을 정면으로 마주하며,  
단순히 작동하는 서비스를 넘어 **‘사용하기 좋은 서비스’를 만드는 개발자**로 성장하는 것을 목표로 한다.

## 프로젝트 배경

CLI 로또 프로그램은 기능적으로는 완성되어 있지만, 사용자 입장에서 UX적 문제가 존재했다:

- 입력 가이드 부족  
- 오류 발생 시 즉시 종료  
- 흐름이 단절됨  
- 시각적 피드백 부족  
- 결과 이해의 어려움  

이를 해결하고자,  
**금액 입력 → 로또 발행 → 당첨 번호 입력 → 결과 확인 → 누적 통계 제공**  
까지의 전체 흐름을 하나의 자연스러운 사용자 경험으로 재설계했다.


## 프로젝트 목표

1. **CLI 기반 로또 프로그램의 웹 서비스화**  
   HTML·CSS·Thymeleaf 프론트엔드와 Java 백엔드를 결합한 완전한 웹 애플리케이션으로 구현한다.

2. **사용자 경험 중심 설계**  
   단순한 웹 포팅이 아니라,  
   **사용자가 쉽게 이해하고 직관적으로 사용할 수 있는 흐름과 인터페이스**를 설계한다.  
   “작동하는 서비스”를 넘어 **“사용하기 좋은 서비스”**를 만드는 것을 목표로 한다.


## 핵심 도전

- **기능 중심 사고에서 사용자 중심 사고로 전환하기**  
  단순히 ‘어떤 기능을 제공할까’를 고민하는 데서 벗어나,  
  **‘사용자가 어떻게 하면 이 기능을 쉽고 즐겁게 사용할 수 있을까’**를 중심으로 사고한다.  
  기능 구현보다 **사용자 경험(UX)**을 우선순위에 두는 개발자의 관점을 길러내는 것이 목표다.

- **디자인이라는 낯선 영역에 도전하기**  
  평소 약점으로 느껴졌던 **디자인 영역**에 직접 도전하고,  
  **UI/UX 디자인 프로세스(리서치 → 디자인 → 피드백 반영)**을 실제 프로젝트에 적용해본다.  
  이를 통해 ‘보기 좋은 화면’을 넘어서, **‘사용하기 좋은 서비스’**를 만들어가는 과정을 경험한다.

- **기존 코드를 웹 서비스로 확장하며 구조적 개선 고민하기**  
  CLI 기반으로 작성된 기존 로또 코드를 웹 서비스로 옮겨가는 과정에서,  
  **코드의 역할과 책임을 재점검하고 더 유연한 구조로 개선하는 방법**을 탐구한다.  
  단순히 작동하는 코드를 넘어, **변화에 강하고 유지보수하기 쉬운 코드**를 지향한다.


## UI/UX 디자인 프로세스 적용

이번 미션에서는 기존 CLI 기반 로또 프로그램을 웹 서비스로 확장하기 위해,  
**Design Council의 Double Diamond 모델**과 **IDEO의 Human-Centered Design 프로세스**를 참고하여 아래와 같은 절차로 진행한다.

### 진행 절차

1. **사용자 관점 이해**  
   사용자가 어떤 상황과 맥락에서 이 서비스를 이용하려 하는지 파악하고,  
   로또 구매 과정에서 궁금해할 정보와 기대하는 경험을 정리한다.

2. **사용자 목표 정의**  
   사용자가 서비스를 통해 이루고자 하는 구체적 목표  
   (예: 빠른 구매, 실수 없는 입력, 결과의 명확한 이해 등)를 도출한다.

3. **기존 서비스 분석**  
   CLI 기반 로또 프로그램이 사용자의 목표를 얼마나 충족하고 있는지 평가하고,  
   웹 서비스로 전환 시 개선이 필요한 요소를 파악한다.

4. **설문 설계 및 분석**  
   사용자에게 직접 궁금한 점을 묻는 설문을 구성하고,  
   결과를 분석해 UX 개선 인사이트를 얻는다.

5. **저니맵 / 정보 구조 설계**  
   사용자의 흐름(구매 → 당첨 확인 → 결과 확인)과  
   각 화면에 필요한 정보를 시각적으로 정리한다.

6. **피그마 디자인**  
   정보 구조와 사용자 흐름을 기반으로 인터페이스를 설계하고,  
   직관적인 화면 구성을 만든다.

7. **프로그래밍 구현**  
   설계된 화면을 HTML, CSS, Thymeleaf, Java로 구현하여  
   실제 웹 서비스 형태로 완성한다.

8. **사용자 테스트**  
   완성된 서비스를 테스트하고,  
   사용성 피드백을 반영해 개선점을 도출한다.
